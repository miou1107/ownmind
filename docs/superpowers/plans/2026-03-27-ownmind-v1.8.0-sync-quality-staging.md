# OwnMind v1.8.0 — Sync Token + 規則品質追蹤 + 暫存區 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 為 OwnMind 加入跨工具狀態一致性（sync token）、規則落地率追蹤（stats + 自評）、Context 40% 自動交接 + 暫存區機制

**Architecture:** Server-side sync token 驗證確保多工具一致性；本地 stats 計數搭便車回填 API；40% context 時自動建立交接並上傳 pending_review 暫存；學習回顧跨 session 整合暫存區 + 智慧過濾

**Tech Stack:** Node.js (Express API + MCP server), PostgreSQL, SHA-256 hash

### Task 1: Sync Token — Server Side

**Files:**
- Create: `src/utils/syncToken.js`
- Modify: `src/routes/memory.js:1-10` (import + SERVER_VERSION)
- Modify: `src/routes/memory.js:228-296` (init route)
- Modify: `src/routes/memory.js:395-428` (POST route)
- Modify: `src/routes/memory.js:433-528` (PUT route)
- Modify: `src/routes/memory.js:534-584` (disable route)
- Modify: `src/routes/memory.js:322-337` (GET type route)

**GIVEN/WHEN/THEN:**

```
GIVEN: 使用者已 init 取得 sync_token
WHEN: 使用者用正確 token 呼叫 POST /api/memory
THEN: 寫入成功，回傳新 token

GIVEN: 使用者已 init 取得 sync_token
WHEN: 另一個工具寫入了新記憶（token 已變）
AND: 使用者用舊 token 呼叫 POST /api/memory
THEN: 回傳 409 + stale: true + new_token

GIVEN: 使用者已 init 取得 sync_token
WHEN: 使用者用舊 token 呼叫 GET /api/memory/type/:type
THEN: 回傳結果 + stale: true + new_token（不阻塞讀取）

GIVEN: 使用者沒有 init（無 token）
WHEN: 呼叫 POST /api/memory
THEN: 回傳 409，強制 init

GIVEN: init 呼叫
WHEN: API 回應
THEN: 包含 sync_token + server_version + team_standards_hash + last_team_standard_update + iron_rules_count

GIVEN: 使用者用正確 token 呼叫 PUT /:id/enable
WHEN: 執行完成
THEN: 記憶恢復啟用，回傳新 token

GIVEN: POST/PUT body 包含 rule_stats 欄位
WHEN: 寫入成功
THEN: server 合併 rule_stats 到對應記憶的 metadata.stats

GIVEN: 使用者從未呼叫 init（無 token）
WHEN: 呼叫 POST /api/memory（body 無 sync_token）
THEN: 回傳 409 + error: '請先呼叫 init 取得 sync token'（與 stale token 區分）
```

- [ ] **Step 1:** 建立 `src/utils/syncToken.js` — generateSyncToken + validateSyncToken
- [ ] **Step 2:** 修改 `src/routes/memory.js` — import syncToken + 定義 SERVER_VERSION
- [ ] **Step 3:** 修改 init route — 回傳 sync_token + version manifest
- [ ] **Step 4:** 修改 POST route — 驗證 token（409 拒絕）+ 回傳新 token
- [ ] **Step 5:** 修改 PUT route — 同上
- [ ] **Step 6:** 修改 PUT disable route — 同上
- [ ] **Step 7:** 修改 GET type route — stale 標記（不阻塞）
- [ ] **Step 8:** 修改 PUT enable route — 加入 sync token 驗證 + 回傳新 token
- [ ] **Step 9:** POST/PUT 收到 `rule_stats` 時，合併到對應記憶的 `metadata.stats`（搭便車 receiver）
- [ ] **Step 10:** 區分 409 錯誤訊息：無 token = `請先 init`；token 過期 = `狀態已變更，請 re-init`

**GIVEN/WHEN/THEN:**

```
GIVEN: MCP client 啟動
WHEN: 呼叫 ownmind_init
THEN: 儲存回傳的 sync_token 到 session 變數

WHEN: 呼叫 ownmind_get
```
